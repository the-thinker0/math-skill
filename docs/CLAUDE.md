# Math Skill 使用指南 / User Guide

## 简介 / Introduction

Math Skill 是一个将数学思维应用到科研和生活的 Claude Code 技能包。包含**十五种数学思想武器**，每个都配有详细的方法论、操作规程和数学出处。

## 安装 / Installation

直接粘贴下面这段给 Claude Code 或其他终端型 AI 助手即可：

```
请帮我安装 math-skill：https://github.com/the-thinker0/math-skill，并教我如何使用
```

如果你更了解自己的环境，也可以指定安装方式（npm / git clone / 本地路径）。

## 使用方法 / Usage

### 手动触发命令

Math Skill 通过特定命令手动触发，**不会在每次对话开始时自动加载**。使用以下命令：

| 命令 | 思想武器 | 适用场景 |
|------|---------|---------|
| `/axiomatization` | 📐 公理化思想 | 审查理论假设、构建理论框架 |
| `/abstraction` | 🧩 抽象化思想 | 提取本质结构、简化复杂问题 |
| `/logic-deduction` | 🧠 逻辑演绎 | 检查论证严谨性、发现逻辑漏洞 |
| `/modeling` | 🌉 建模思想 | 构建模型、预测现象 |
| `/optimization` | ⚖️ 优化思想 | 资源分配、决策优化 |
| `/probability-statistics` | 🎲 概率与统计 | 量化不确定性、数据分析 |
| `/transformation` | 🔄 变换思想 | 换个角度、简化难题 |
| `/symmetry-invariance` | ⚛️ 对称与不变性 | 寻找不变量、分类研究对象 |
| `/induction-analogy` | 📈 归纳与类比 | 从特殊到一般、跨领域借鉴 |
| `/algorithmic-thinking` | 🖥️ 算法与计算思想 | 判断可行性、设计求解步骤 |
| `/information-theory` | 📡 信息论思想 | 量化不确定性、评估信息价值 |
| `/game-theory` | 🎯 博弈论思想 | 多方互动决策、机制设计 |
| `/causal-inference` | 🔗 因果推断思想 | 区分因果与相关、评估干预效果 |
| `/topological-thinking` | 🌀 拓扑思想 | 连通结构分析、形状分类 |
| `/discrete-combinatorial` | 🧮 离散与组合思想 | 计数、枚举、发现有限结构规律 |

### 使用示例 / Examples

```
# 科研场景：审查论文的理论基础
/axiomatization 这篇论文声称从三个假设推导出了一个新定理，但我怀疑它的假设集合可能不自洽...

# 科研场景：检查证明的逻辑
/logic-deduction 这篇论文的定理 3.2 的证明中，从第 5 行到第 6 行的推导看起来有跳跃...

# 科研场景：建模实验数据
/modeling 我做了一组实验，发现变量 x 和 y 之间有某种关系，但不知道用什么模型来描述...

# 科研场景：优化实验设计
/optimization 我有 3 个实验要做，但只有 2 周时间，每个实验需要不同设备和准备时间，怎么安排最高效？

# 科研场景：统计检验
/probability-statistics 我的实验结果显示新药有效率比旧药高 15%，p = 0.03，样本量 n=50，这个结果可靠吗？

# 生活场景：时间分配
/optimization 我需要在论文写作、课程作业和社交之间分配时间，论文 deadline 还有两周...

# 生活场景：决策分析
/probability-statistics 新闻说某种食物能使患癌风险降低 30%，我应该多吃吗？

# 生活场景：换个角度
/transformation 我一直无法解决和室友的矛盾，在当前的沟通方式下僵局似乎无法打破...
```

### 语言切换 / Language Switching

**默认输出语言为中文。** 如需英文输出，在命令后追加 "in English"：

```
/optimization How should I allocate my research time? in English
```

### 组合使用 / Combining Skills

多个思想武器可以组合使用：

```
# 先用建模思想构建模型，再用概率统计来验证
/modeling 先帮我建立一个预测模型...
/probability-statistics 然后用统计方法检验这个模型的预测效果...

# 先用抽象化找到结构，再用优化思想求解
/abstraction 这个问题的本质结构是什么？
/optimization 在这个抽象框架下，最优策略是什么？
```

## 十五思想武器速查 / Quick Reference

### 📐 公理化思想

从最少假设出发，通过严格逻辑构建理论体系。用于审查论文假设的合理性、发现理论内部矛盾。

### 🧩 抽象化思想

抓住本质特征，忽略非本质细节。用于简化复杂问题、发现不同领域的共性。

### 🧠 逻辑演绎

从真命题出发，严格推理新真命题。用于检查证明严谨性、识别逻辑谬误。

### 🌉 建模思想

现实问题 → 数学问题 → 解释现实。用于构建科研模型、规划生活决策。

### ⚖️ 优化思想

约束条件下寻找最优解。用于资源分配、多目标取舍。

### 🎲 概率与统计

量化不确定性，从数据提取规律。用于实验数据分析、风险评估、避免认知偏差。

### 🔄 变换思想

复杂问题 → 等价的简单问题。用于换角度看问题、简化难题。

### ⚛️ 对称与不变性

寻找变化中的不变性质。用于揭示隐藏规律、分类研究对象。

### 📈 归纳与类比

从特殊到一般，从已知到未知。用于提出假说、跨领域借鉴。

### 🖥️ 算法与计算思想

将问题转化为可执行步骤，分析计算代价，判断可行性。用于自动化流程、评估NP-hard问题。

### 📡 信息论思想

信息是不确定性的减少。用于量化不确定性、最优压缩、信道容量分析。

### 🎯 博弈论思想

最优策略取决于他人的选择。用于多方互动决策、机制设计、竞争分析。

### 🔗 因果推断思想

相关≠因果，但因果可形式化。用于干预效果评估、区分混淆变量、反事实推理。

### 🌀 拓扑思想

连续变形下不变的性质。用于连通结构分析、拓扑数据分析、形状分类。

### 🧮 离散与组合思想

计数、枚举、发现有限对象的规律。用于系统性枚举、鸽巢推理、生成函数分析。

## 知识体系 / Knowledge Base

`knowledge-base/overview.md` 提供了完整的数学知识体系概述，包括：

- 三大基础支柱（代数、几何、分析）
- 主要分支（数论、概率统计、拓扑、离散数学、应用数学）
- 知识层次（地基、代数、综合、前沿）
- 与其他学科的联系

## 数学审视 / Math Critic

`agents/math-critic.md` 定义了一个子 agent，兼具审视与实现能力。审视维度与十五种数学思想武器一一对应，但不机械遍历——根据问题性质选择最相关的3–5个维度深入检查。审视发现问题后，选择合适的思维武器给出具体实现方案（推导步骤、证明框架、算法设计或模型改进），而非仅指出问题。
